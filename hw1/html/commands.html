More neat commands

In the demonstration that I provided we got to see several of steghides commands in action.Steghide however is not limited to those commands and functionalities. For example we did not see that steghide has a command to display all of the encryption algorithms that they support. As you can see below they support many different encryption algorithms all which can be chosen with a different command. 


          ┌──(Julio㉿DESKTOP-STHA33J)-[/mnt/c/Users/Julio/Pictures/steghide_example]
          └─$ steghide --encinfo
          encryption algorithms:
          <algorithm>: <supported modes>...
          cast-128: cbc cfb ctr ecb ncfb nofb ofb
          gost: cbc cfb ctr ecb ncfb nofb ofb
          rijndael-128: cbc cfb ctr ecb ncfb nofb ofb
          twofish: cbc cfb ctr ecb ncfb nofb ofb
          arcfour: stream
          cast-256: cbc cfb ctr ecb ncfb nofb ofb
          loki97: cbc cfb ctr ecb ncfb nofb ofb
          rijndael-192: cbc cfb ctr ecb ncfb nofb ofb
          saferplus: cbc cfb ctr ecb ncfb nofb ofb
          wake: stream
          des: cbc cfb ctr ecb ncfb nofb ofb
          rijndael-256: cbc cfb ctr ecb ncfb nofb ofb
          serpent: cbc cfb ctr ecb ncfb nofb ofb
          xtea: cbc cfb ctr ecb ncfb nofb ofb
          blowfish: cbc cfb ctr ecb ncfb nofb ofb
          enigma: stream
          rc2: cbc cfb ctr ecb ncfb nofb ofb
          tripledes: cbc cfb ctr ecb ncfb nofb ofb

In the previous example we could have chosen to encode using any of these encryption methods. To select which encryption -e followed by our choice would have been used. We could have also chosen to hide the embed files name by using -N. Quickly I will show those two in action by running the following command. 


          ┌──(Julio㉿DESKTOP-STHA33J)-[/mnt/c/Users/Julio/Pictures/steghide_example]
          └─$ steghide embed -cf ferrari.jpg -ef sample-data.csv -sf steg_ferrari2.jpg -p mysecretpass -e "blowfish cbc" -N
          embedding "sample-data.csv" in "ferrari.jpg"... done
          writing stego file "steg_ferrari2.jpg"... done

I have created a new file by the name of steg_ferrari2.jpg which is distinct and separate from the steg_ferrari.jpg from the previous demonstration. I also used -N so that the name of the embedded file is kept secret and I used -e to set the encryption method to blowfish cbc. This is what all three files look like side by side. Visually they are indistinguishable.

Then by using the info command and entering in the correct passphrase we can see that the name of the hidden file is not given to us and that blowfish cbc encryption was used. 


          ┌──(Julio㉿DESKTOP-STHA33J)-[/mnt/c/Users/Julio/Pictures/steghide_example]
          └─$ steghide --info steg_ferrari2.jpg
          "steg_ferrari2.jpg":
            format: jpeg
            capacity: 16.2 KB
          Try to get information about embedded data ? (y/n) y
          Enter passphrase:
            embedded data:
              size: 4.6 KB
              encrypted: blowfish, cbc
              compressed: yes

These are a few more commands and there are many more to explore. Funny enough you can actually learn about more commands by running the “man steghide” command. By reading through this documentation you can learn about commands that allow you to choose between compressing or not as well as at what compression level to embed your files. You can also read more directly on their github[1].
